*A note on identifier policy*

Spotify track, artist, and playlist IDs are public identifiers and not required for interpreting/analyzing results. My suggestion is to treat them as internal values and omit them in public releases.

---

# Raw Feature Data
This directory contains raw data collected from the Spotify Web API. Files are generated by `data_collection.ipynb`.

**playlist_tracks_YYYY-MM-DD.csv:** playlist membership on a given date.

You should see the following columns.
* `playlist_id`
* `track_id`
* `track_name`
* `artist_id`
* `artist_name`
* `added_at`
* `snapshot_date`

---

**playlist_followers_YYYY-MM-DD.csv**: playlist follower counts on today's date.


**Columns**
* `playlist_id`
* `playlist_name`
* `followers`
* `snapshot_date`

---

**tracks_features.csv:** track metadata used for feature engineering.

**Columns**
* `track_id`
* `track_name`
* `track_popularity` - popularity score (0–100)
* `duration_ms`
* `explicit`
* `release_date` - date as provided by Spotify
* `release_date_precision` - year, month, day
* `album_id`
* `album_name`
* `artist_id`
* `artist_name`

---

**artists_features.csv:** artist metadata associated with tracks in the dataset.

**Columns**
* `artist_id`
* `artist_name`
* `artist_popularity`
* `artist_followers`
* `genres`
