*A note on identifier policy*

Spotify track, artist, and playlist IDs are public identifiers and not required for interpreting/analyzing results. My suggestion is to treat them as internal values and omit them in public releases.

---

# Processed Feature Data
This directory contains processed feature tables used for modeling playlist growth and recommendation behavior. All files are generated by `feature_engineering.ipynb`.

---

**track_features_processed.csv:** track feature matrix

You should see the following columns.
* `track_popularity`
* `artist_popularity`
* `log_artist_followers`
* `track_age_days`
* `explicit`

**Genre Features**
* Multi-hot encoded genre columns (`genre_*`)
* Each column represents presence (1) or absence (0) of genre tag

---

**playlist_features.csv**: playlist features from track metadata

**Columns**
* `playlist_id`,
* `track_popularity_mean`,
* `track_popularity_median`,
* `track_popularity_std`
* `artist_popularity_mean`
* `log_artist_followers_mean`
* `track_age_days_mean`
* `track_age_days_median`
* `explicit_mean`
*  several different kinds of genres
* `playlist_size`
* `num_unique_artists`
* `artist_diversity_ratio`
